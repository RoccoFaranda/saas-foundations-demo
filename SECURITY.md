# Security Policy

## Supported Scope

Security reports are welcome for the current live deployment and the latest state of the `main` branch.

## Reporting a Vulnerability

Do not open a public GitHub issue or pull request for security findings.

Send a private report to `support@saasfoundationsdemo.com` with the subject line `Security report: SaaS Foundations Demo`.

Include:

- a clear description of the issue
- affected URLs, routes, or files
- reproduction steps
- impact assessment
- any proof-of-concept material needed to verify the report

## Response Expectations

Responses are handled on a best-effort basis for a solo-maintained portfolio project. Private disclosure is preferred until the issue has been reviewed and, where applicable, remediated.
